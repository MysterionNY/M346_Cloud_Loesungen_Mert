## CloudFormation

### Was ist CloudFormation?

AWS CloudFormation ist ein Infrastructure-as-Code (IaC) Service. Er ermöglicht die Definition von AWS-Ressourcen in YAML oder JSON Templates. Diese Templates werden als "Stacks" deployed und alle Ressourcen werden automatisch erstellt und verwaltet.

### CloudFormation vs Cloud-Init

| Aspekt | CloudFormation | Cloud-Init |
|--------|----------------|------------|
| **Scope** | Gesamte AWS-Infrastruktur (EC2, VPC, RDS, LB, etc.) | Nur eine einzelne EC2-Instanz |
| **Zeitpunkt** | Vor/während Ressourcen-Erstellung | Nach EC2-Start (First Boot) |
| **Aufgabe** | AWS-Ressourcen erstellen und verwalten | Betriebssystem konfigurieren |
| **Format** | YAML/JSON Templates | YAML (cloud-config) |
| **Beispiel** | "Erstelle EC2 + Load Balancer + RDS" | "Installiere nginx, erstelle User" |

**Zusammenspiel:** CloudFormation erstellt die EC2-Instanz und übergibt Cloud-Init Konfiguration. Cloud-Init konfiguriert dann das OS beim ersten Start.

### Automatisch erstellte Objekte (Elastic Beanstalk)

Folgende Ressourcen wurden von Elastic Beanstalk automatisch über CloudFormation erstellt:

| Ressource | Typ | Beschreibung |
|-----------|-----|--------------|
| EC2 Instanz(en) | AWS::EC2::Instance | Application Server |
| Security Group (EC2) | AWS::EC2::SecurityGroup | Firewall für Instanzen |
| Security Group (LB) | AWS::EC2::SecurityGroup | Firewall für Load Balancer |
| Application Load Balancer | AWS::ElasticLoadBalancingV2::LoadBalancer | Traffic-Verteilung |
| Target Group | AWS::ElasticLoadBalancingV2::TargetGroup | Instanz-Registrierung |
| Listener | AWS::ElasticLoadBalancingV2::Listener | Port 80 HTTP |
| Auto Scaling Group | AWS::AutoScaling::AutoScalingGroup | Automatische Skalierung |
| Launch Configuration | AWS::AutoScaling::LaunchConfiguration | EC2 Konfiguration |
| CloudWatch Alarms | AWS::CloudWatch::Alarm | Monitoring & Alerts |

### Vergleich KN06 (IAAS) vs KN07 (PAAS)

| Objekt | KN06 (IAAS) | KN07 (PAAS/Elastic Beanstalk) |
|--------|-------------|-------------------------------|
| EC2 Instanzen | Manuell erstellt | Automatisch erstellt |
| Security Groups | Manuell konfiguriert | Automatisch erstellt |
| Load Balancer | Manuell erstellt | Automatisch erstellt |
| Target Groups | Manuell erstellt | Automatisch erstellt |
| Auto Scaling | Manuell eingerichtet | Automatisch konfiguriert |
| Cloud-Init | Selbst geschrieben (YAML) | Nicht nötig |
| Deployment | Manuell (git clone, systemctl) | Ein Klick (Upload ZIP) |
| Updates | Manuell | Managed Updates möglich |

**Hauptunterschiede:**
1. Bei PAAS werden alle Infrastruktur-Komponenten automatisch erstellt
2. Kein Cloud-Init Script nötig - Elastic Beanstalk konfiguriert alles
3. Deployment ist einfacher (Upload statt SSH + git)
4. Skalierung und Monitoring sind integriert

**Fazit:** PAAS (Elastic Beanstalk) nimmt die gesamte Infrastruktur-Verwaltung ab. Bei IAAS (KN06) musste alles manuell konfiguriert werden - EC2, Security Groups, Load Balancer, Auto Scaling und Cloud-Init Scripts.
